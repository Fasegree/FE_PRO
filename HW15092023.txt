Как отказаться от изменений в файле (файлах) в рабочем каталоге?
git checkout -xdn  для отмены изменений в конкретном файле.

Как вернуть файл из индекса в рабочий каталог?
git restore --staged 

Как отменить ошибочный commit?
git reset HEAD^^ 
git reset HEAD~2
(откатить на 2 коммита)

Как исправить коммит, отправленный на сервер GitHub?
git push --force может перезаписать историю ветки и привести к потере данных.


Как создать новую ветку и переключиться в нее?
git checkout -b <имя новой ветки>

В чем отличие между fast-forward merge и non-fast-forward merge?
Fast-forward merge происходит, когда история изменений в ветке, которую хотим слить, линейна и можно просто продвинуть указатель текущей ветки вперед до указателя другой ветки. Non-fast-forward merge происходит, когда история изменений нелинейна и требуется создание нового коммита слияния для объединения изменений из двух веток.

Когда может возникнуть конфликт слияния веток, как его разрешить?
Конфликт слияния возникает, когда Git не может автоматически определить, как объединить изменения из разных веток. Для разрешения конфликта нужно вручную отредактировать конфликтующие файлы, удалить метки конфликта (<<<<<<<, =======, >>>>>>>) и создать новый коммит. После этого используем git add и git commit для завершения слияния.

Можно ли удалить ветку? Как?
git branch -d <имя ветки>  ( -D для принудительного удаления)
